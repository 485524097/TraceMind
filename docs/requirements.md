# TraceMind 第一阶段需求说明

## 项目背景

中文开发者常将技术文档、项目代码、数据库脚本和问题记录分散保存在多个位置，检索、理解和复用成本较高。TraceMind 希望以本地优先的方式沉淀这些资料，并让 AI 回答具备可核验的来源。

## 目标用户

- 需要整理个人技术资料的中文开发者
- 需要快速理解存量项目与历史问题的工程师
- 准备以可演示作品展示工程能力的求职开发者

## 核心问题

1. 技术资料和代码分散，难以统一管理。
2. 关键词检索无法充分理解自然语言问题与语义关联。
3. AI 问答缺少来源，难以验证答案可信度。
4. 历史问题和解决经验难以持续复用。

## 第一阶段目标

第一阶段重点场景是：

> 导入一个包含 Java、JSP、JavaScript、SQL、Markdown 和 PDF 的项目资料库，用户可以通过关键词或自然语言查询代码、业务逻辑和历史问题，并获得带文件路径、页码或代码行号的回答。

当前交付已覆盖知识库管理、原始文件本地持久化、增量版本管理，以及 Markdown、UTF-8 文本、代码、PDF 文本层和 DOCX 的异步解析与可追溯 Chunk。检索索引与问答仍属于后续交付。

## MVP 功能范围

- 知识库、文件和导入任务的基础管理
- Markdown、PDF 及常见代码文件的解析与分块
- 全文检索、向量检索、混合检索与重排
- 基于知识库的问答与对话历史
- 文件路径、页码、章节和代码行级引用
- 多模型配置与检索效果评测

当前文档导入约束：只允许配置中的常见技术文件扩展名，按同知识库规范化文件名识别逻辑文档，按 SHA-256 判断内容是否变化；不同文件名不跨文档去重。

当前解析约束：文本和代码仅支持 UTF-8/UTF-8-SIG；PDF 仅提取文本层且不支持 OCR；代码按行切分而非 AST；`parse_status=succeeded` 只表示 Chunk 可用，不表示已建立检索索引。

## 暂不实现的功能

当前 Dense 语义检索阶段明确暂不实现 BM25、Sparse Vector、Hybrid Search、Reranker、RAG 和 LLM Answer。

- 完整知识图谱
- 多用户权限系统
- 手机客户端
- 插件市场
- 复杂 Markdown 编辑器
- 自动研究 Agent
- 自动个人画像
- 音频和视频解析

## 核心使用场景

1. 导入一个包含 Java、JSP、JavaScript、SQL、Markdown 和 PDF 的项目资料库。
2. 通过关键词定位代码、表结构、设计文档和历史问题。
3. 使用自然语言提问业务逻辑或代码关系。
4. 查看回答对应的文件路径、PDF 页码、章节或代码行号并回溯原文。
5. 沉淀开发问题与解决经验，供后续项目检索复用。

## 安全和隐私约束

- 仅导入拥有授权的数据、代码和文档。
- 不提交 API Key、密码、客户数据、公司内部资料或其他私人敏感信息。
- 所有配置应通过环境变量管理；`.env.example` 可以纳入版本控制，但真实 `.env` 文件不得提交。
- 优先保持资料和服务本地可控，引用必须来自已导入文档和代码的元数据。

## 秋招展示目标

- 展示从资料导入、解析、检索到带引用问答的完整闭环。
- 展示对 Java 项目资料、文档与历史问题的联合检索能力。
- 展示清晰的工程分层、可验证引用和可复用的开发经验沉淀。

## 第一阶段 RAG 边界

第一条问答链路为单轮、无状态的 Citation-grounded Streaming RAG。Dense Retrieval 是
唯一召回方式；无足够相关来源时不调用 LLM。回答提供 `[Sx]` 与原始 Chunk 定位，但
当前不实现对话历史、Agent、Tools、Hybrid Search 或 Reranker。
